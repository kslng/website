Intro to KSL

KSL is a language designed to be strongly typed, aot-compiled, and memory safe. Along with some other neat features like function attributes and pretty much whatever else seems worthy of having.

In this guide we'll briefly discuss how to get started programming your first project in KSL. Of course, the first thing you'll want to do is download the KSL compiler from the home page.

Next create a KSL file. You can do this with either .k or .ksl, for now the file name itself doesn't matter much. Let's get it open in your favourite code editor now.

Just like in other lower level programming languages, your program will need an entry point. For this let's create a main() function. 
function main() -> null {
	return;
}
Alright, there's a lot going on there so let's talk about it. function clearly states that we're starting a new function, then the main that comes after that is the name of said function. Don't worry about the open and close parentheses for now. The little -> symbol is what tells KSL the expected return type for the function, for our main function, we said the expected return type should be null, or effectively nothing.

Great, now we have a .k or .ksl file that has a function in it called main. Time to get this baby compiled! Open up your command line and use the following command:
ksl <your_file_name>
Alternatively, just run ksl to see a list of options, build flags, and whatnot.

At the time of this page being written KSL doesn't have a linking process built in, which means that it will return an unlinked object file that will need to be manually linked with a different compiler or linker.
